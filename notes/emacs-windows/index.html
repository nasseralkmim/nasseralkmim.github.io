Emacs on windows with WSL2

Jan 4 2021

Introduction

Build emacs inside windows subsystem for linux (wsl). The main advantage is that emacs is considerably faster when using LSP.

Reference for installation

GitHub - hubisan/emacs-wsl: Install and run emacs with the Windows Subsystem â€¦

Shortcut

One .bat file that starts the vcxsrv is it nor already running and calls the power shell script that starts emacs. 

@echo off

TASKLIST | FINDSTR vcxsrv || start "" "C:\Program Files\VcXsrv\vcxsrv.exe" :0 -multiwindow -clipboard -wgl -ac

@rem Just run the PowerShell script.

powershell -NoProfile -ExecutionPolicy Bypass -Command "& './wsl-emacs.ps1'"

One power shell script, provided by the Hubisan repository. Just added the keyboard layout and key chord to switch.

# WSL2 - Launch Emacs

# Get the IP to use from wsl and set to a variable.
$wslip = wsl -d Ubuntu-20.04 bash -c 'ip route | awk ''/default via /'' | cut -d'' '' -f3'

# Run Emacs
wsl -d Ubuntu-20.04 bash -c "export DISPLAY=$wslip`:0.0 export LIBGL_ALWAYS_INDIRECT=1 && setxkbmap -layout 'br,us' -option grp:shifts_toggle && setsid emacs"

Then, a .vbs file that launches the bat script.

If WScript.Arguments.Count <= 0 Then
    WScript.Quit
End If

bat = Left(WScript.ScriptFullName, InStrRev(WScript.ScriptFullName, "\")) & WScript.Arguments(0) & ".bat"
arg = ""

If WScript.Arguments.Count > 1 Then
    arg = WScript.Arguments(1)
End If

CreateObject("WScript.Shell").Run """" & bat & """ """ & arg & """", 0, False

And finally a shortcut to this vbs with target the .bat script

C:\Windows\System32\wscript.exe C:\Users\nasse\OneDrive\emacs-wsl\run-emacs-wsl-2\launcher.vbs "wsl-emacs"

Fonts

Download the .ttf font file, move them to ~/.fonts/ and fc-cache -f -v to rebuild font cache.

LaTeX

TeX Live is a easy way to have TeX programs, macro packages and fonts to produce documents with LaTeX macros.

sudo apt install tex-common texlive-base texlive-latex-recommended \
                 texlive-latex-extra texlive-fonts-recommended latex-cjk-all

To install biblatex we need sudo apt install biber.

To view pdf using Windows SumatraPDF add folder with SumatraPDF.exe on windows $PATH (environment variables) and this to init.el.

(setq TeX-view-program-list
	  '(("Sumatra PDF" ("\"SumatraPDF.exe\" -reuse-instance"
			    (mode-io-correlate " -forward-search %b %n ") " %o"))))
    (assq-delete-all 'output-pdf TeX-view-program-selection)
    (add-to-list 'TeX-view-program-selection '(output-pdf "Sumatra PDF"))

To get inverse search using wsl emacsclient -n +%l "$(wslpath '%f')" on SumatraPDF options.

To get the fonts that I use sudo apt install texlive-fonts-extra.

Hugo

I had problems with the Ubunto 20.04 version of hugo. The solution is to just use the windows version of Hugo and call it within wsl with hugo.exe instead of just hugo.

DONE Emacs jupyter

It seems that emacs-jupyter was looking for jupyter in the search path that emacs uses to find programs (exec-path variable).

jupyter --path gives:

config:
    /home/nasser/.jupyter
    /usr/etc/jupyter
    /usr/local/etc/jupyter
    /etc/jupyter
data:
    /home/nasser/.local/share/jupyter
    /usr/local/share/jupyter
    /usr/share/jupyter
runtime:
    /home/nasser/.local/share/jupyter/runtime

Which is not inside exec-path variable. Added with (add-to-list 'exec-path "/usr/etc").

I also was getting error for emacs-jupyter not finding python (apply: Searching for program: No such file or directory, python). Emacs-jupyter was not finding python, even though I could call python3 in the wsl terminal. It seems that emacs-jupyter (jupyter-locate-python) looks for python only at jupyter --paths.

More specifically emacs-jupyter looks at /usr/local/bin/ and my python3 was installed which python3 at /usr/bin/. So I had to create a symbolic link sudo ln -s /usr/bin/python3 /usr/local/bin/python. Then it worked.

DONE Treemacs visit-node-in-external-application

This is a reason for me to keep standard emacs on windows.

When I call this function in a .sh file it opens as a text in another emacs instance. However I want it to just run, currently I have to call treemacs-run-shell-command-for-current-node and manually type the filename ./script.sh.

For instance, in windows I just go to deploy.sh on treemacs and hit o x to run the script.

Use wsl-open to fix that. GitHub - 4U6U57/wsl-open: Open files with xdg-open on Bash for Windows in Winâ€¦

Change the default application that runs shell scripts (application/x-shellscript) for the wsl-open. Use xdg-mime query filetype file.sh to discover the filetype. And use xdg-mime query default <new filetype> to discover which application is the default for it. Currently is emacs.desktop, which opens the shell script in another emacs instance. I created a wsl-open.desktop inside ~/.local/share/applications with 

[Desktop Entry]
Name=wsl-open
Exec=/usr/local/bin/wsl-open
Type=Application

and set as default with xdg-mime default wsl-open.desktop application/x-shellscript. Now when I open a shell script with xdg-open (x o in treemacs) it runs the script just like in windows.
