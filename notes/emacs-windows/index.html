Emacs on windows with WSL2

Jan 4 2021

Reference for installation

GitHub - hubisan/emacs-wsl: Install and run emacs with the Windows Subsystem …

Shortcut

I had to use two files:

first one is a .vsb with

If WScript.Arguments.Count <= 0 Then
    WScript.Quit
End If

bat = Left(WScript.ScriptFullName, InStrRev(WScript.ScriptFullName, "\")) & WScript.Arguments(0) & ".bat"
arg = ""

If WScript.Arguments.Count > 1 Then
    arg = WScript.Arguments(1)
End If

CreateObject("WScript.Shell").Run """" & bat & """ """ & arg & """", 0, False

and the second a .bat script.

u/ECHO OFF
TASKLIST | FINDSTR XWwin || START /B C:\cygwin64\bin\run.exe --quote /usr/bin/bash.exe -l -c "XWin :0 -listen tcp -multiwindow -clipboard -wgl -ac"
wsl zsh -c "export DISPLAY=$( awk '/nameserver/ { print $2 }' /etc/resolv.conf ):0; export LIBGL_ALWAYS_INDIRECT=1; eval $(dbus-launch); export DBUS_SESSION_BUS_ADDRESS; setxkbmap -layout "br,us" -option grp:shifts_toggle; setsid emacs;"

Notice:

  1. keyboard layout set to br and us with two shifts as toggle shortcut.

Finally the shortcut to the .vbs

C:\Windows\System32\wscript.exe C:\Users\nasse\Documents\emacs-wsl-shortcut\bat-launcher.vbs ""script"

Hugo

Apt in wsl2 has only an old version of hugo. This version does not properly process org-mode dates format, so I was getting 01/01/0001 as a date.

I had to manually download hugo Install Hugo | Hugo binary and put it in usr/local/bin.

Also, another problem, it seems that when hugo server it does not detect changes to update the html rendering. microsoft/WSL#4739 {WSL2} File changes made by Windows apps on Windows filesy… I don't have a solution for this yet.

Trying the solution in here WSL 2 installation and changes. Problems and solutions before and after… | by…

Emacs jupyter

Emacs-jupyter was not finding python, even though I could call python in the wsl terminal. It seems that emacs-jupyter was looking for python at limited places, namely jupyter --path.

I had to create a symlink for python3 to one of jupyter --path so emacs-jupyter can find python.

Treemacs visit-node-in-external-application

When I call this function in a .sh file it opens as a text in another emacs instance. However I want it to just run, currently I have to call treemacs-run-shell-command-for-current-node and manually type the filename ./script.sh.

For instance, in windows I just go to deploy.sh on treemacs and hit o x to run the script.

Org-download not getting clipboard from windows

It does not work with emacs in WSL2. It seem xclip can not get clipboard.

  1. microsoft/WSL#4440 WSL2 clipboard not shared between Linux and Windows

Uninstalling zsh

When I uninstalled zhs from wsl, it wouldn't start up again.

This blog post helped. Diagnosing and Fixing WSL initialization issues | HildenCo Software

I had to go to go to System32 folder, log in wsl as a root wit

C:\Windows\System32>wsl -u root

then change the default shell to bash again.

chsh -s /bin/bash name
