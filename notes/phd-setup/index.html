Emacs

Emacs, of course.

Tried briefly overleaf, it was good first. Nice to have instant previews and so on.

But then, it got little complicated with the bibliography export from Zotero. If I add a reference to zotero I have to manually update the bibliography, and it also takes some time to update. Real downside.

Another minor problem that I anticipated is when I create figures with scripts, I would have to manually upload then (? I think so). With emacs, they are already on drive and all changes automatically update on the document.

Bibliography

Zotero for managing the references and creating the .bib file.

My workflow typically is:

  1. insert the doi of the reference I want to read/cite

  2. automatically import it into my .bib with Better Bibtex

  3. try to find the pdf, also automatically

Problems with zotero:

  1. I don't like the random naming of the storage folders. It creates an unwelcoming dependence on the software. I would rather have the author/year convention. But it is fine since I don't like to keep tons of unused pdf's around. From time to time I delete them but keep the zotero for the .bib file.

I just tried an emacs package called ebib and it seems to be good for inserting citations in latex.

Better yet, bibtex-actions. Only problem is that I can not open the pdf, not sure if the problem is with windows path or something else. But this is better because of embark flexibility.

Notes on CMake, make and makefile

  1. make is a tool that controls the generation of executables

    1. make compiles a program from source code

    2. make compile a final executable binary that can be installed using make install

    3. it enables to build and install a package without knowing the details of it, make knows which files need to be updated

  2. a makefile is a file required by make and defines a set of tasks to be executed

    1. without a makefile one needs to type clang -o <executable> <file1.c> <file2.c> .... every time to compile the executable

    2. if you change only 1 .c file, you would have to recompile all of them which is inefficient, just run make is better

  3. CMake is a tool for build, test and package software

  4. CMake is cross platform, it recognizes which compilers to use and automatically discovers and configure the tool chain to compile the source code in any operating system

  5. CMake is equivalent to running ./configure (check if all tools needed are in place)

  6. CMakeList.txt has information for cmake

  7. CMake control software compilation process and generate "native makefiles" and "workspaces"

    1. native makefiles and workspaces can be used in the "compiler environment"

References:

Fortran compiler

MPI

Message passing interface. MPI is a standardized API for parallel computing, FAQ: General information about the Open MPI Project.

sudo apt install mpi

BLAS and LAPACK

Third part libraries (TLPs) that are required.

sudo apt install libblas-dev liblapack-dev

Configuration script

I created a shell script with in a folder with Trilinos source code cloned with git clone https://github.com/trilinos/Trilinos/

cmake \
-DTPL_ENABLE_MPI=ON \
-DMPI_BASE_DIR=./mpi/ \
-DTrilinos_ENABLE_ALL_PACKAGES=ON \
-DCMAKE_INSTALL_PREFIX=./trilinos-build/ \
./Trilinos/
