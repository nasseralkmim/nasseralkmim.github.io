Zotero

Better bibtex

<2021-05-26 Wed> Better to let it automatically update so to avoid potential errors. I got a Zotero error and it was solved by updating beter-bibtex.

This is an essential plug-in. It allows creating a .bib file that we can use to insert references in a .tex document. It also auto updates the file when another reference is inserted in Zotero.

So the workflow is: import a reference in Zotero with the DOI link and use emacs bibtex-actions to insert the citation in the .tex document.

Backup

Zotero creates a folder with all its data and automatically stores a local copy of the pdfs in its folder structure. So you don't need to keep the pdf file in a different directory.

Zotero sync can save some of the data but the pdf file storage capacity is very limited. The preferably way to backup the library is to copy the whole Zotero directory.

To restore just the data, bibtex entries for instance, Zotero sync is enough. I think keeps all the pdfs is excessive, and not necessary.

One downside is that if you want look for a pdf you have to use Zotero application because its folder structure is not human readable.

ZotFile

<2021-06-26 Sat>: Who needs multiple collections and categories when you have "orderless" narrowing completion. By having a tailored file link to each bibtex entry, the default behavior of Zotero, it makes loading the bibliography slow in bibtex-completion.el (part of helm-bibtex).

<2021-05-15 Sat>: I don't care about pdfs since I can get them whenever I need. The downside is that I don't keep the notes that I usually write on the pdf. But this can be circumvented by always writing relevant information in a proper review file.

Zotero calls library data the collection of titles, authors, citation key and so on. Where as the pdf are called attached files.

Zotero has two forms of handling pdf attachments: default storage and linked files.

The default storage uses the same Zotero data directory to store and manage the pdfs attachments. The advantage is you can manage it through the Zotero app. So you can rename the files automatically or delete them, for instance.

The default storage is good if you use syncing to also store files attachments. This way you can have the attachments in multiple machines. However the free version is limited to 300mb.

If you don't sync files attachments you are not going to be able to access the files attachments in a different machine.

Linked files Zotero stores only a link to the original pdf. So you have to manually manage where to store the original pdf and their original name. I would keep all of them in a single cloud folder.

The ZotFile plug-in can be useful to manage those linked files.

ZotFile renaming and moving files to specific folder

Within ZotFile preferences, we can:

  1. Location of Files and custom location specify where to save the .pdf and link them to the reference item inside Zotero.

  2. Renaming rules and renaming format we can specify the format {%b} which is the same for the bibtex key.

    1. this is convenient for searching a file using just the key and a storage location.

    2. this way we can just keep a single folder with all the pdfs and use emacs orderless with bibtex-completion to filter and narrow the list of references with (i) independent terms (author, year, title) in (ii) any order.

note: the terms do not need to be exact, they could be partial. e.g, to search "Numerical … by Trefethen … in 1997" we can type num so we can narrow the list the items with "Num". Then type 97 so we get the ones with "97". Finally, if this was not enough, and we vaguely remember that the author starts with "T" we can type T to further narrow down the options for completion. Really efficient.

Transporting the pdf

  1. start Zotero in the new machine

  2. sync library data with Zotero account

  3. copy storage folder that contains the files into the newly created Zotero data directory
